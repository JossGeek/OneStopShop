OneStopShop — Scraper & Import Architecture

Two decoupled APScheduler jobs share a CrawlUrl queue table. The Crawler discovers URLs; the Scraper processes them. Manual imports write directly via the API.

Reflects logic in backend/content/scrapers/service.py, source_registry.py, and content/ingestion/importer.py

Job 1 — Crawler · every 360 min

URL Discovery → CrawlUrl Queue

Runs for each configured source. Populates the queue; never fetches page content.

  A(["Crawler job triggered"]) --> B["Load enabled sources from source_registry"]

  I --> J

Job 2 — Scraper · every 5 min

URL Processing → Offer Upsert

Claims a batch of pending/due CrawlUrl rows and processes each one. Creates a ScrapingRun record with full structured log.

flowchart TD
  A(["Scraper job triggered"]) --> B["Claim up to BATCH_SIZE\npending/due CrawlUrls\n(status = processing)"]
  B --> C{"URLs claimed?"}
  C -- "No" --> Z1(["Done — queue empty or all future-scheduled"])
  C -- "Yes" --> D["Create ScrapingRun (RUNNING)"]

  D --> E["For each URL"]

  E --> F["Fetch HTML\n(timeout + UA header)"]
  F --> G{"Fetch OK?"}

  G -- "No (network / 5xx)" --> ERR1["Increment consecutive_errors"]
  ERR1 --> ERR2{"errors ≥ MAX?"}
  ERR2 -- "Yes" --> ERR3["status = archived\nArchive/delete linked offer"]
  ERR2 -- "No"  --> ERR4["status = error\nBackoff: 1h → 6h → 24h"]
  ERR3 --> NEXT
  ERR4 --> NEXT

  G -- "404 / 410" --> GONE["status = archived\nArchive/delete linked offer"]
  GONE --> NEXT

  G -- "2xx" --> H["Deterministic extraction\n(BeautifulSoup / JSON-LD / meta)"]

  H --> I{"crawl_enabled source?"}

  I -- "Yes" --> J["AI relevance check + extraction\n(Ollama model pool)"]
  J --> K{"Relevant offer page?\nconfidence ≥ threshold?"}
  K -- "No" --> NEG["Decision: NEGLECT\nLog reason code\nstatus = done (skipped)"]
  K -- "Yes" --> L["Use LLM result if confidence\n≥ deterministic, else keep det."]
  NEG --> NEXT

  I -- "No (direct URL)" --> M["AI is primary extractor\nDeterministic is fallback"]
  M --> L

  L --> N["Upsert Offer (get_or_create + update)\nLink CrawlUrl.offer\nstatus = done\nnext_check_at = now + 7 days"]
  N --> NEXT

  NEXT["Log url_processed / url_neglected entry\nto ScrapingRun.log"] --> MORE{"More URLs\nin batch?"}
  MORE -- "Yes" --> E
  MORE -- "No" --> FINALIZE["Update ScrapingRun counters\n(created / updated / unchanged /\nneglected / errors)\nStatus = success / failed"]

  FINALIZE --> STALE["Flag stale candidates\n(successful sources only)"]
  STALE --> CLEANUP["Cleanup broken links\n(skip URLs fetched OK this cycle)"]
  CLEANUP --> Z2(["ScrapingRun complete\nStats returned"])

API — Manual Import

CSV / Excel → Offer + CrawlUrl

Operator uploads a file to preview, reviews validation output, then confirms. No scraper job involved.

CrawlUrl Status Lifecycle

stateDiagram-v2
  [*] --> pending : Crawler discovers URL\nor import confirms row

  pending --> processing : Scraper claims batch

  processing --> done      : 2xx — offer upserted or page skipped\nnext_check_at = now + 7 days
  processing --> error     : 5xx / timeout (consecutive_errors < MAX)\nbackoff: 1h → 6h → 24h
  processing --> archived  : 404 / 410 or consecutive_errors ≥ MAX\nlinked offer archived / deleted

  error --> pending : backoff expires → re-queued

  done --> pending : revisit window expires (7 days)
Decision codes logged per URL:
mapped — offer upserted
neglected / non_relevant_page — crawl page rejected
neglected / missing_core_fields — extraction incomplete
failed — HTTP or network error
archived — permanent 404/410 or repeated errors
