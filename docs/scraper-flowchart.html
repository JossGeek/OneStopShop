OneStopShop Scraper Workflow

Flowchart of how the scraper runs, processes each source, handles errors, and cleans invalid links.

Based on current logic in content/scrapers/service.py

    flowchart TD
      A(["Start scrape cycle"]) --> B["Init ScrapeService"]
      B --> C["Load source list"]
      C --> D{"Sources found?"}
      D -- "No" --> E(["Return empty summary"])
      D -- "Yes" --> F["Load scraping type + bot user"]

      F --> G["For each source"]
      G --> H["Sync job + create RUNNING run"]

      H --> I["Process source"]
      I --> I1["Fetch HTML (timeout + UA)"]
      I1 --> I2["Deterministic extraction"]
      I2 --> I3{"Confidence low or missing fields?"}
      I3 -- "Yes" --> I4["Run Ollama fallback"]
      I4 --> I5["Pick better payload"]
      I3 -- "No" --> I5
      I5 --> I6["Upsert offer + refresh scraping metadata"]

      I6 --> J["Run SUCCESS + counters + log"]
      I --> K{"Exception raised?"}
      K -- "Yes" --> L["Run FAILED + error log"]
      L --> M{"HTTP status 404 or 410?"}
      M -- "Yes" --> N["Delete offers for invalid source"]
      M -- "No" --> O["Keep failed run record"]
      N --> O
      J --> P["Finalize run + update next_run_at"]
      O --> P
      P --> Q{"More sources?"}
      Q -- "Yes" --> G
      Q -- "No" --> R["Flag stale candidates (successful sources only)"]

      R --> S["Cleanup broken links for scraping offers
skip links fetched successfully in this cycle"]
      S --> T(["Return summary stats"])

      classDef decision fill:#fff7db,stroke:#b45309,stroke-width:1px,color:#7c2d12;
      classDef fail fill:#fee2e2,stroke:#b91c1c,stroke-width:1px,color:#7f1d1d;

      class A,E,T start;
      class B,C,F,G,H,I,I1,I2,I4,I5,I6,J,N,O,P,R,S process;
      class D,I3,K,M,Q decision;
      class L fail;
Legend: green = start/end, blue = processing, yellow = decisions, red = failure path. The cleanup step ensures offers with broken links (HTTP 404/410) are removed.
